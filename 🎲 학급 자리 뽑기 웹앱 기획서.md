🎲 학급 자리 뽑기 웹앱 기획서

1. 프로젝트 개요

항목내용프로젝트명학급 자리 뽑기 (ClassSeat Picker)대상중·고등학교 교사 및 학생플랫폼웹 브라우저 (모바일/PC 반응형)기술 스택HTML + CSS + Vanilla JavaScript (단일 파일)


2. 핵심 기능 목록

2-1. 학생 명렬 관리


학생 이름 직접 입력 (한 명씩 추가)
이름 목록 붙여넣기 (엑셀/메모장에서 복사 → 한 번에 등록)
학생 삭제 / 전체 초기화
명렬 저장 (localStorage 활용, 브라우저 재접속 후에도 유지)


2-2. 교실 좌석 설정


행(Row) × 열(Column) 수 입력으로 좌석 배치도 자동 생성
특정 좌석 비활성화 (비어있는 자리, 교사석 등)
좌석 배치도 미리보기


2-3. 뽑기 모드 (3가지)

🎯 모드 1 — 랜덤 배치


버튼 한 번으로 전체 학생을 좌석에 무작위 배정
결과 표 형태로 즉시 출력
재뽑기 버튼으로 반복 가능


🎡 모드 2 — 룰렛 뽑기


빈 좌석을 클릭하면 룰렛이 회전하며 학생 이름 결정
한 명씩 순차적으로 자리를 채워가는 방식
회전 애니메이션 + 효과음(선택)으로 몰입감 제공


🃏 모드 3 — 카드 뽑기


학생 이름이 적힌 카드가 뒤집어진 상태로 화면에 배열
교사(또는 학생 본인)가 카드를 클릭하면 뒤집히며 배정 좌석 공개
플립 애니메이션 적용


2-4. 결과 출력


좌석 배치표를 화면에 표시 (인쇄 최적화 CSS 포함)
PNG 이미지로 저장 (html2canvas 활용)
텍스트로 복사 (클립보드)

4. UI/UX 설계 원칙


단일 페이지 구성: 스크롤 또는 탭 전환으로 단계 이동, 로딩 없음
모바일 우선: 터치 조작에 최적화, 버튼 크기 최소 44px
접근성: 고대비 색상, 폰트 크기 조절 가능
피드백: 뽑기 실행 시 시각적 애니메이션 + 간단한 사운드 효과
실수 방지: 재뽑기 전 확인 다이얼로그 표시

6. 개발 단계 (로드맵)

단계내용예상 시간1단계명렬 관리 + 좌석 설정 UI2h2단계랜덤 배치 기능 + 결과 출력1h3단계룰렛 뽑기 애니메이션3h4단계카드 뽑기 플립 애니메이션2h5단계저장/인쇄/이미지 내보내기1h6단계반응형 디자인 + 테스트1h

총 예상 개발 시간: 약 10시간


7. 확장 아이디어 (추후 추가 고려)


성별 분리 배치: 남녀 혼합/분리 옵션
조건부 배치: 특정 학생은 앞자리 고정 (시력 배려 등)
히스토리 관리: 이전 배치 기록 저장 및 비교
모둠 뽑기 모드: 좌석 배치가 아닌 팀 구성 뽑기
QR 공유: 결과를 QR 코드로 학생들과 공유
다국어 지원: 영어 등 추가 언어 지원
